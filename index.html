🎯 The Problem We're Solving

Data teams spend 40-60% of their time fighting fires: investigating pipeline failures, tracking down data quality issues, and manually writing validation rules. This reactive approach leads to delayed detection, manual diagnosis, repetitive work, and inconsistent quality.

  • Delayed detection: Issues are found downstream, after business impact
  • Manual diagnosis: Engineers spend hours digging through logs and metrics
  • Repetitive work: Writing similar validation rules over and over
  • Inconsistent quality: Each team has different standards and tools

📖 The Solution: DRE 3D Framework

DRE 3D is a self-driving data reliability engine that combines AI agents with Databricks platform to automate the entire data quality lifecycle. Built 100% on Databricks native components.

🏗️ How It Works: The Three Layers

The system is organized into three layers, each with clear responsibilities. Think of it like a car: the dashboard (frontend), the engine (RAG/AI), and the wheels (execution). Frontend/Orchestration → RAG/Knowledge → Execution (D1/D2/D3)

Layer 1: Frontend & Orchestration

Users interact with Databricks Apps to write rules or ask questions. LangGraph orchestrates multi-agent conversations (Parser → Validator → Security → SQL Generator → Approval)

📄 PUML Source 🖼️ PNG 📐 SVG

Layer 2: RAG Knowledge Base

AI agents retrieve relevant context from documentation, past incidents, and logs. Uses UC Volumes for raw docs, Delta for chunked data, and AI Search for semantic retrieval with CRAG confidence gating.

📄 PUML Source 🖼️ PNG 📐 SVG

Layer 3: Execution (D1/D2/D3)

Databricks Workflows run scheduled detection, diagnosis, and deflection jobs. D1 detects anomalies → D2 diagnoses root cause → D3 generates fixes and creates tickets.
